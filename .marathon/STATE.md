**Phase 4, Unit 3: mtime tracking that feeds `{changedFiles}` to `findReusablePriorBlock`.** Goal: given a prior block's `relevant_files` (string paths, relative to `cwd`), determine which have CHANGED so the early-exit can refuse a stale reuse. Implement a helper — keep it pure-ish and testable by injecting a `statImpl`/clock — e.g. `computeChangedFiles(relevantFiles, sinceMs, {cwd, statImpl})` → `string[]` of paths considered changed. A path counts as **changed** if: it no longer exists (stat throws/ENOENT — deleted/moved), OR its `mtimeMs > sinceMs`. `sinceMs` = the timestamp of the prior block (the turn it was produced). **PROBLEM: prior blocks currently carry NO timestamp** — `extractPriorRLMBlocks` returns `{intent, relevant_files, raw}`.
